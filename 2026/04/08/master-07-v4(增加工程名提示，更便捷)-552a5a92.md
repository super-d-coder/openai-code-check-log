### 代码评审

**错误提示**：存在潜在的空指针异常。
**错误位置**：`resolveProjectName` 方法内的 `Paths.get(System.getProperty("user.dir")).getFileName().toString()` 行。
**修改建议**：`Path.getFileName()` 可能返回 `null`（例如当路径为根目录时），直接调用 `toString()` 会抛出 `NullPointerException`。建议增加非空判断。
**修改后的代码片段**：
```java
    private String resolveProjectName() {
        String repository = System.getenv("GITHUB_REPOSITORY");
        if (!isBlank(repository)) {
            return repository.trim();
        }

        // 修改部分开始
        Path path = Paths.get(System.getProperty("user.dir"));
        Path fileNamePath = path.getFileName();
        String dirName = fileNamePath != null ? fileNamePath.toString() : null;
        // 修改部分结束

        return isBlank(dirName) ? "unknown-project" : dirName;
    }
```

### 总结
代码逻辑清晰，但在获取本地目录名时未处理 `getFileName()` 为 null 的边界情况，可能导致程序崩溃，建议按上述方案修复后合入。
